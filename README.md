Public showcase for AI Sandbox engineering capability with strict confidentiality boundaries.

## Purpose

This repository is a public entry point to demonstrate team integration capability,
SDK delivery discipline, and governance quality without exposing proprietary internals.

## Included

- Capability map across public SDKs
- Security boundary and disclosure rules
- Public-safe integration guidance
- Automated security and disclosure signals (Actions + templates)

## Not Included

- Proprietary architecture internals
- Private endpoints, prompts, model assets, or customer data
- Credentials, tokens, or infrastructure identifiers

## Governance Signals

- Security policy: `SECURITY.md`
- Contribution standards: `CONTRIBUTING.md`
- Ownership rules: `.github/CODEOWNERS`
- Issue intake template: `.github/ISSUE_TEMPLATE/bug_report.md`
- Pull request disclosure template: `.github/pull_request_template.md`
- Automated checks: `.github/workflows/public-security-guards.yml`
- Dependency maintenance: `.github/dependabot.yml`
