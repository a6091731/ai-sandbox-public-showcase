# Security Boundary

## Boundary Statement

This showcase demonstrates technical capability while protecting product and
customer confidentiality.

## Publicly Shareable

- API/SDK integration patterns
- CI/release/governance practices
- De-identified engineering outcomes

## Restricted

- Proprietary architecture internals
- Private endpoints, credentials, and infrastructure identifiers
- Prompt assets, model configuration, and customer-identifying data

## Pre-Publish Checks

1. No secrets or private links
2. No customer-identifying details
3. No proprietary implementation disclosure
4. Dual technical and security review completed
